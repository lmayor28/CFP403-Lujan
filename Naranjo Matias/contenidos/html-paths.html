Guía para Entender Rutas (Paths) en HTML

¿Qué es una Ruta (Path)?

En desarrollo web, una "ruta" o "path" es la dirección que especifica la ubicación de un archivo (una página HTML, una imagen, un archivo CSS, un script de JavaScript, etc.) dentro de la estructura de carpetas de un sitio web o sistema de archivos.

Cuando creas enlaces (`<a href="...">`), insertas imágenes (`<img src="...">`), vinculas hojas de estilo (`<link href="...">`) o scripts (`<script src="...">`), necesitas usar rutas para decirle al navegador dónde encontrar esos recursos.

Tipos de Rutas

Rutas Absolutas

Una ruta absoluta proporciona la ubicación completa y exacta de un archivo, independientemente de dónde se encuentre el archivo actual que la contiene. Son como direcciones postales completas.

Se usan principalmente para:

  • Vincular a recursos en **otros sitios web** en internet. Comienzan con el protocolo (http://, https://).
  • Referenciar archivos desde la **raíz del sitio web** (ver rutas relativas con `/`).

Ejemplo de ruta absoluta a un sitio externo:

<a href="https://www.google.com">Ir a Google</a>

Ejemplo de ruta absoluta a una imagen alojada en otro servidor:

<img src="https://www.ejemplo-externo.com/images/logo.png" alt="Logo Externo">

Rutas Relativas

Una ruta relativa especifica la ubicación de un archivo **en relación con la ubicación del archivo HTML actual** que la contiene. Son muy útiles para vincular archivos dentro de tu propio proyecto porque funcionan sin importar en qué dominio o carpeta de tu computadora esté el proyecto, siempre que la estructura interna se mantenga.

./ (Punto Barra) - Directorio Actual

El símbolo ./ se refiere al **directorio (carpeta) actual** donde se encuentra el archivo HTML desde el que estás creando el enlace o referencia.

Es la forma más simple de vincular a otro archivo que está en la **misma carpeta** que el archivo actual.

Ejemplo basado en tus archivos:

Si tienes:

- mi-proyecto/
  - index.html    <-- Estás aquí
  - pagina2.html
  - pagina3.html
  - style.css
  - logo.png

Desde index.html, para vincular a pagina2.html (que está en la misma carpeta), usas:

<a href="./pagina2.html">Ir a Página 2</a>

Para vincular al archivo CSS o una imagen en la misma carpeta:

<link rel="stylesheet" href="./style.css">
<img src="./logo.png" alt="Mi Logo">

Aunque a menudo puedes omitir el ./ (href="pagina2.html" también funciona si están en el mismo nivel), usarlo es más explícito y claro para entender que la ruta es relativa al directorio actual.

../ (Punto Punto Barra) - Directorio Padre

El símbolo ../ significa "subir **un nivel** en la estructura de directorios" (ir a la carpeta inmediatamente superior o "padre" de la carpeta actual).

Lo usas para salir de la carpeta actual y acceder a archivos o carpetas que están "por encima" o en una rama diferente del árbol de directorios.

Ejemplo:

Si tuvieras esta estructura:

- mi-proyecto/
  - paginas/
    - index.html    <-- Estás aquí
    - pagina2.html
  - css/
    - estilos.css
  - images/
    - logo.png
  - sobre.html

Desde paginas/index.html:

  • Para ir a sobre.html (que está en la carpeta principal mi-proyecto/): 
    <a href="../sobre.html">Sobre Nosotros</a>
    (Subes un nivel de paginas/ a mi-proyecto/, y ahí encuentras sobre.html)
  • Para vincular a estilos.css (dentro de la carpeta css/ en el nivel superior): 
    <link rel="stylesheet" href="../css/estilos.css">
    (Subes un nivel a mi-proyecto/, luego entras en la carpeta css/, y ahí encuentras estilos.css)
  • Para vincular a logo.png (dentro de la carpeta images/ en el nivel superior): 
    <img src="../images/logo.png" alt="Logo del Sitio">
    (Subes un nivel a mi-proyecto/, luego entras en la carpeta images/, y ahí encuentras logo.png)

Puedes encadenar ../ para subir múltiples niveles (ej: ../../ sube dos niveles).

/ (Barra al principio) - Raíz del Sitio Web (Contexto Web)

Cuando una ruta relativa comienza con una barra / (sin un punto antes), no se refiere a la raíz de tu sistema de archivos, sino a la **raíz del sitio web** o dominio desde el que se está sirviendo la página.

Es útil para referenciar archivos que sabes que siempre estarán en un lugar fijo con respecto a la raíz de tu sitio (por ejemplo, una carpeta css o js en la raíz).

Ejemplo (asumiendo que la carpeta `mi-proyecto` es la raíz de tu sitio web):

- mi-proyecto/ <-- Raíz del sitio ( / )
  - index.html
  - paginas/
    - contacto.html  <-- Estás aquí (ruta relativa paginas/contacto.html)
  - css/
    - style.css      <-- Ruta relativa css/style.css
  - js/
    - script.js      <-- Ruta relativa js/script.js

Desde cualquier archivo (como `contacto.html` dentro de `paginas/`), para vincular a `style.css` o `script.js` usando rutas relativas a la raíz del sitio:

<link rel="stylesheet" href="/css/style.css">  <-- Siempre busca css/style.css desde la raíz del sitio
<script src="/js/script.js"></script>         <-- Siempre busca js/script.js desde la raíz del sitio

Esto es útil porque la ruta `/css/style.css` funciona sin importar si estás en `index.html` (raíz) o en `paginas/contacto.html` (subcarpeta).

Resumen

  • **Ruta Absoluta:** Dirección completa, principalmente para enlaces externos (http://...).
  • **Ruta Relativa (./):** En la misma carpeta que el archivo actual.
  • **Ruta Relativa (../):** Subir un nivel de carpeta.
  • **Ruta Relativa (/ al principio):** Desde la raíz del sitio web (en contexto web).

Entender y usar correctamente las rutas relativas es clave para construir sitios web navegables y portátiles.
