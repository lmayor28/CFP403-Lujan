Controlando Qué Ignorar con .gitignore

¿Por qué usar .gitignore?

En cualquier proyecto de desarrollo, hay archivos y carpetas que no quieres que Git rastree. Esto incluye:

  • Archivos temporales generados por tu editor o sistema operativo (.DS_Store, archivos de copia de seguridad .bak).
  • Archivos generados automáticamente por procesos de compilación (.class, .o, .pyc).
  • Dependencias de proyecto descargadas (la carpeta node_modules/ en Node.js, paquetes de Python, etc.).
  • Archivos de configuración local que contienen información sensible o específica de tu entorno (claves API, configuraciones de base de datos locales).

El archivo .gitignore le indica a Git qué archivos y directorios debe ignorar. Es una parte importante de un repositorio limpio y colaborativo.

Cómo Crear y Usar .gitignore

Simplemente crea un archivo de texto llamado .gitignore en la **raíz de tu repositorio Git** (la carpeta principal de tu proyecto).

Dentro de este archivo, escribes patrones que coinciden con los archivos o carpetas que quieres ignorar.

Patrones Comunes

Cada línea en .gitignore es un patrón:

  • **Ignorar archivos por extensión:** 
    *.log     # Ignora todos los archivos con extensión .log
*.temp    # Ignora todos los archivos .temp
*.bak     # Ignora todos los archivos .bak
*.pyc     # Ignora archivos compilados de Python
  • **Ignorar un archivo específico por nombre:** 
    mi-archivo-de-prueba.txt  # Ignora solo este archivo
  • **Ignorar todos los archivos con un nombre específico en cualquier directorio:** 
    archivo-config.ini # Ignora cualquier archivo llamado archivo-config.ini
  • **Ignorar una carpeta completa:** Agrega una barra / al final. 
    node_modules/  # Ignora la carpeta node_modules y todo su contenido
build/         # Ignora la carpeta build y todo su contenido
  • **Ignorar una carpeta específica en la raíz del repositorio:** Agrega una barra / al principio y al final. 
    /temp/   # Ignora la carpeta 'temp' solo si está en la raíz
  • **Ignorar un archivo con un nombre específico solo en la raíz:** Agrega una barra / al principio. 
    /.env    # Ignora el archivo .env solo si está en la raíz
  • **Negar un patrón (no ignorar un archivo/carpeta que de otra forma sería ignorado):** Usa ! al principio. 
    # Ignorar todos los archivos .log
*.log
# PERO, no ignores el archivo importante.log
!importante.log

**Recuerda:**

  • Las reglas de .gitignore solo afectan a los archivos que **no están siendo rastreados** por Git. Si ya agregaste y confirmaste un archivo, Git lo seguirá rastreando aunque luego lo añadas a .gitignore. Para que Git lo ignore, primero debes dejar de rastrearlo (usando git rm --cached nombre_archivo) y luego hacer un commit.
  • Debes añadir y confirmar el archivo .gitignore en sí mismo para que sus reglas se apliquen a otros colaboradores si compartes tu repositorio.

Existen plantillas de .gitignore muy útiles en línea (por ejemplo, en GitHub) para diferentes lenguajes y frameworks que puedes usar como punto de partida.
