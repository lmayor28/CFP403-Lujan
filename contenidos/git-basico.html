Git: Fundamentos del Control de Versiones

¿Qué es Git?

Git es un **sistema de control de versiones distribuido** gratuito y de código abierto diseñado para manejar todo, desde proyectos pequeños hasta muy grandes, con velocidad y eficiencia. Fue creado por Linus Torvalds (el creador de Linux).

La idea principal es que Git te ayuda a rastrear y registrar los cambios que haces en tus archivos a lo largo del tiempo, como una serie de "instantáneas" o "commits". Esto te permite:

  • Ver el historial completo de tu proyecto.
  • Saber exactamente qué cambios se hicieron, quién los hizo y cuándo.
  • Revertir a versiones anteriores si algo sale mal.
  • Trabajar en diferentes características o ideas en paralelo sin afectar la versión principal del proyecto.

Conceptos Clave y Comandos Básicos

Repositorio Git

Es la base de tu proyecto en Git. Es una carpeta que contiene todos tus archivos y la base de datos interna de Git (la carpeta .git/ oculta) que rastrea el historial de cambios.

Puedes inicializar un nuevo repositorio en una carpeta existente:

git init

O clonar uno existente desde un servicio como GitHub:

git clone [URL_del_repositorio]

Área de Preparación (Staging Area)

Antes de guardar los cambios en un commit, primero los "preparas" o "stages". Es como una canasta donde pones los archivos que quieres incluir en tu próximo punto de guardado.

Usas git add para añadir cambios al área de preparación:

git add nombre_del_archivo.html  # Agrega un archivo específico
git add .                     # Agrega todos los cambios en la carpeta actual y subcarpetas

Commit

Un commit es una "instantánea" oficial de tu proyecto en un momento dado. Es un punto en el historial al que puedes volver. Cada commit debe tener un mensaje descriptivo.

Usas git commit para guardar los cambios que están en el área de preparación:

git commit -m "Mensaje descriptivo del commit"

El mensaje debe ser conciso (primera línea) y puede tener un cuerpo más detallado después de una línea en blanco.

Estado del Repositorio

Conocer el estado de tu repositorio es fundamental. Te dice qué archivos han sido modificados, cuáles están en el área de preparación y cuáles no están siendo rastreados.

Usas git status para ver el estado:

git status

Te mostrará cosas como "Changes to be committed" (en staging), "Changes not staged for commit" (modificados pero no añadidos al staging) y "Untracked files" (archivos nuevos que Git aún no sigue).

Git es una herramienta muy poderosa. Dominar estos comandos básicos (`init`, `add`, `commit`, `status`) es el primer paso para gestionar tus proyectos de manera efectiva y prepararte para colaborar.
